Architecture

The Architecture of AI-LINK

  1. Clients: Clients initiate the process by performing local model training on their devices (Step 1) using their private data. This data is never shared directly; instead, an encrypted hash of the data is created and submitted to the smart contract on blockchain (Step 2). Then, clients submit their encrypted data to the server directly (Step 3).
  2. Server: The server requests the local model proof from the smart contract (Step 4), and upon receiving acknowledgment (Step 5), it begins training a global model using the submitted encrypted data (Step 6). Once the global model is trained, a hash proof of the global model is submitted back to the smart contract (Step 7).
  3. Blockchain and Smart Contract: The blockchain acts as the mediator and verifier. It receives encrypted data hash proof from the clients (Step 2) and manages the proof requests and acknowledgments between clients and the server. Once the global model is verified, it is distributed to the clients (Step 8), who can request and acknowledge the global model proof (Steps 9 and 10).
